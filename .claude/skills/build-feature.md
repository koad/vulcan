Standard build workflow for assigned work.

## Steps

1. Read the issue fully — understand acceptance criteria before touching code
2. Check what already exists — don't rebuild what's there
3. Implement the minimum that satisfies the criteria
4. Test it works as expected
5. Commit with a message that references the issue number
6. Push and comment on the issue with what was done and how to verify

## Principles

- Solve the issue as scoped. Don't add unrequested features.
- If the issue is unclear, comment asking for clarification before building.
- If blocked, say so on the issue immediately — don't go silent.
